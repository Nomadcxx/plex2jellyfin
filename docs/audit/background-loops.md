## Needs Deeper Research

1. **`service.CleanupService` vs `Consolidator` vs `Executor`+`Planner`** — Three systems for duplicate detection/resolution. `CleanupService` is used by housekeeping's `detectFileDuplicates` and `execConsolidateDuplicate`. `Executor`+`Planner` uses `consolidation_plans` DB table. `Consolidator` uses `conflicts` table. Need to trace which CLI commands and daemon paths use which system to confirm the `Consolidator` is truly dead code. Check: `cmd/` directory for CLI commands referencing `consolidate.Consolidator`.

2. **`isAllDigits`/`allDigits`/`isAllDigitsOrPunct`** — These string-validation helpers appear in housekeeping but may also exist in `naming` or `database` packages. A cross-package grep for `func isAllDigits\|func allDigits\|func isAllDigitsOrPunct` would confirm whether a shared utility already exists.

3. **`isEmptyDir` / `removeDirIfEmpty` / `removeIfEmptyTree` / `CleanupEmptyDir`** — Directory-empty utilities scattered across housekeeping and consolidate. Check if `internal/paths` or another shared package already has these.

4. **`formatBytes`** — Check if `internal/format` or similar package already has a bytes-to-human function before moving it.

5. **`executeDelete` triplication** — The 3 delete implementations (plans.go, executor.go, housekeeping.go) differ in permission-checking and dry-run handling. Need to verify whether a unified `DeleteMediaFile` can satisfy all callers without losing safety checks (permissions.go integration in plans.go's version).
