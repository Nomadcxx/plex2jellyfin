## Needs Deeper Research

1. **`DeferredQueue.GetAll()` / `PlaybackLockManager.GetAll()` / `Count()` usage**: These methods may only be used in tests and API status endpoints. If the API status endpoint is the only production consumer, consider whether a simpler `Len()` or direct map access suffices. Cross-reference: `internal/api/`, `internal/daemon/`.

2. **`PluginClient` vs `Client` merge**: `PluginClient` duplicates ~80% of `Client`. The only difference is retry logic in `request()`. Could `Client` gain a `WithRetry(bool)` option, eliminating `PluginClient` entirely? Check all `PluginClient` call sites in `internal/organizer/` to confirm no other behavioral differences.

3. **`isMediaFile` consolidation**: 4 copies across `cmd/jellywatch/main.go`, `cmd/jellywatchd/main.go`, `internal/consolidate/consolidate.go`, `internal/jellyfin/verify.go`. Some take `ext string`, some take `path string`. Consolidating requires agreeing on signature and placement. `internal/mediautil` or similar shared package.

4. **Radarr context-wrapper pattern scope**: The `*Context` wrapper pattern infects `client.go`, `commands.go`, `movies.go`, `queue.go`, `config.go`. Total ~120 lines of wrappers. Verify no external caller depends on the non-Context variants before deleting.

5. **`MetadataClient`/`MetadataRepairClient`/`MetadataStore` interface removal impact**: These interfaces exist only for test mocking. Tests in `metadata_recovery_test.go` use a mock store. Check if the mock can be a concrete struct implementing the same methods without the interface declaration, or if tests can use a real `*Client` with `httptest.Server`.

6. **`joinURL` dedup placement**: Both `sonarr/url.go` and `radarr/url.go` have identical `joinURL`. A shared `internal/arrutil` package would work, but adds a new package for 8 lines. Alternative: inline `url.Parse` + `ResolveReference` at each call site (it's 3 lines of actual logic). Weigh package overhead vs duplication.

7. **`MetadataRecoveryConfig` name collision**: `config/config.go` and `jellyfin/metadata_recovery.go` both define `MetadataRecoveryConfig` with different fields. The config one is user-facing (mapstructure tags); the jellyfin one is internal. Renaming one avoids confusion but requires updating all references in `cmd/jellywatchd/main.go` and tests.
